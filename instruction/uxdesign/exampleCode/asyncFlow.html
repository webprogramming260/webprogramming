            What Each Box Means

            Call Stack

            The call stack is where JavaScript runs code frames right now. It executes one frame at a time, and async callbacks can only run when the stack is clear.

            Web APIs

            Web APIs handle browser-managed async work such as timers, network requests, and response parsing. They run outside the JS stack and signal completion later.

            Macrotask Queue (tasks)

            This queue stores completed task callbacks like setTimeout handlers. The event loop moves one task to the call stack only after the stack is empty.

            Microtask Queue (promises)

            This queue stores promise continuations from await and then/catch/finally. Microtasks run before the next task, immediately after current stack work finishes.

            Render Loop (frames)

            DOM updates can queue during JS execution, but visual paint happens on frame ticks after stack and microtasks are clear.
